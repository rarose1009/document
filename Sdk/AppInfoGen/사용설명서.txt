AppInfoGen 사용 설명서

1. AppInfoGen은 무엇에 쓰는 물건인가?
   ; 이것은 왜 사용을 하는가? 어플리케이션 팝파일을 만들면, 어플리케이션이 
   ; 설치되고, 어플리케이션 매니저를 통하여 각 어플리케이션이 관리를 받게 된다.
   ; 그때 필요한 파일이, AMInfo와 AppInfo가 있는데, AMInfo에 등록된 어플리케이션 리스트를 통해
   ; 여러개의 어플리케이션들을 어플리케이션 매니저는 관리하게 되고, 각 어플리케이션은 AppInfo를 통해
   ; 접근이 가능하다. 어떤 특정한 어플리케이션 팝파일이 생성 되었다면, 그 팝파일로부터 간단히
   ; AMInfo와 AppInfo를 생성해 낼 수 있다. 그 작업을 AppInfoGen를 통해 할 수 있다. 

2. 왜 AppInfoGen을 가지고 AMInfo와 AppInfo를 만들어야 하나?
   ; 사실상 어플리케이션을 설치하는 설치 툴들이 모든 작업을 해 주기 때문에, 직접 이 파일들을 생성할 필요는
   ; 거의 없다. 그런데, 매번 설치툴을 이용하지 않고, 간단히 이 파일들을 생성하고, 단말기에 복사를 하거나
   ; 데스크 탑에서 디버그를 할때 유용하기 때문에 종종 사용할 필요성은 있다. 그래서 없애지 않았다.

3. 어떻게 쓰지?
   ; 사용법은 입산해서 수련을 닦을 필요 까진 없을 것 같다.
   ; 때론 사람에 따라서 다르겠지만...
   ; 아래와 같다.
   ; 먼저, 어플리케이션 팝파일을 만든다.
   ; 예를 들어, sample.pop 파일이 만들어 졌다고 가정하면,

   ; 도스모드 창에서 아래와 같이 친다.

   ; C:/>AppInfoGen sample.pop 2502

   ; 맨뒤의 번호는 디바이스의 고유 버전이다. 물론 엠팝에서 사용되는 것이다.
   ; 2502는 아마도 IPAQ디바이스를 알려주는 거다. 아님 말구.

   ; 엔터키를 치고 나면 결과가 나오고, 작업 디렉토리에 AMInfo와 AppInfo가 생성되어 있을 것이다.
   ; 없다면 그냥 포기하고 집에 가시오.
